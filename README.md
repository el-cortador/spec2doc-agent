# spec2doc-agent

Локальный ИИ-агент, который конвертирует постановки аналитиков (PDF, DOCX) в структурированные черновики технической документации в формате Markdown.

Вся обработка происходит на локальной машине — никакие данные не покидают устройство.

---

## Возможности

- Загрузка PDF и DOCX через веб-интерфейс (drag-and-drop или диалог выбора файлов)
- Пакетная обработка нескольких файлов с отображением статуса и оценкой оставшегося времени
- Генерация черновика через локальную LLM (Ollama + `qwen3:4b`)
- Автоматическое определение типа постановки: нарратив, User Story, Jira/Confluence, смешанный
- Структурированный вывод: Описание, Предварительные условия, Порядок выполнения, Ограничения, Параметры конфигурации, Результат
- Предпросмотр черновика прямо в браузере с рендерингом Markdown
- Сохранение результата в `output/{имя_файла}_черновик доки.md`
- Индикатор доступности Ollama и модели в шапке страницы

## Установка и запуск

### 1. Установить Ollama

Скачать и установить с [ollama.com](https://ollama.com), затем загрузить модель:

```bash
ollama pull qwen3:4b
```

### 2. Клонировать репозиторий и установить зависимости

```bash
git clone <url-репозитория>
cd spec2doc-agent
pip install -r requirements.txt
```

### 3. Запустить приложение

```bash
python run.py
```

Скрипт сам запустит Ollama (если не запущена), дождётся её готовности и поднимет веб-интерфейс.

Откройте в браузере: [http://localhost:5000](http://localhost:5000)

> Если Ollama уже запущена в другом терминале — `run.py` это определит и не создаст второй процесс.

> Для остановки нажмите **Ctrl+C** — Ollama и Flask завершатся вместе.

## Использование

1. Убедитесь, что индикатор в шапке страницы зелёный («Ollama подключена · qwen3:4b готова»)
2. Перетащите один или несколько PDF/DOCX-файлов в зону загрузки (или нажмите «Выбрать файлы»)
3. Нажмите «Обработать»
4. Дождитесь статуса «Готово» напротив каждого файла
5. Нажмите «Просмотр» для предпросмотра черновика в браузере
6. Готовые файлы сохраняются в папку `output/` рядом с `main.py`

### Ограничения загрузки

- Поддерживаемые форматы: `.pdf` (с текстовым слоем), `.docx`
- Максимальный размер файла: 20 МБ
- Сканированные PDF (без текстового слоя) не поддерживаются

## Альтернативный бэкенд: llama.cpp

Если Ollama недоступна, можно использовать llama.cpp-сервер с OpenAI-совместимым API:

```bash
BACKEND=llamacpp python run.py
```

По умолчанию ожидается адрес `http://localhost:8080`. Сервер должен быть запущен с моделью `qwen3:4b` или совместимой.

## Структура проекта

```
spec2doc-agent/
├── main.py                 # Flask-приложение: маршруты
├── run.py                  # Единая точка запуска (Ollama + Flask)
├── requirements.txt
├── app/
│   ├── config.py           # Константы: пути, лимиты, модель, URL
│   ├── jobs.py             # Job-датакласс, очередь задач, воркер, save_draft
│   ├── parser.py           # Извлечение текста из PDF и DOCX
│   └── generator.py        # Формирование промпта, вызов Ollama / llama.cpp
├── prompts/
│   └── system_prompt.md    # Системный промпт агента (логика трансформации)
├── static/
│   └── marked.min.js       # Рендеринг Markdown (локальная копия, без CDN)
├── templates/
│   └── index.html          # Веб-интерфейс
├── tests/
│   ├── test_app.py
│   ├── test_generator.py
│   └── test_parser.py
├── uploads/                # Временные входные файлы (создаётся автоматически)
└── output/                 # Готовые черновики .md (создаётся автоматически)
```

## API

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/` | Веб-интерфейс |
| `POST` | `/upload` | Загрузка файлов (`multipart/form-data`, поле `files[]`) |
| `POST` | `/process` | Запуск обработки (`{"job_ids": [...]}`) |
| `GET` | `/status?job_id=` | Статус задачи + время начала и средняя длительность |
| `GET` | `/preview?job_id=` | Содержимое готового черновика |
| `GET` | `/health` | Доступность Ollama и модели |

**Возможные значения `status`:** `queued` / `processing` / `done` / `error`

## Настройка логики трансформации

Поведение агента определяется файлом `prompts/system_prompt.md`. Замена этого файла меняет логику генерации без правки кода.
